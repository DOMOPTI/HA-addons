# Public documentation

Interface contracts and behavior documentation. May be read by agents working in other repositories. This is your component's public API surface in narrative form.

## Authorship rules

- Written for tools, agents, and developers who are NOT working inside this repo.
- No implementation details. Move those to internal/.
- Examples must be runnable against the documented public interface only.
- When a public contract changes, document it here. When you remove something, deprecate first.

Start with these, add more as needed: interfaces.md, architecture.md, behavior.md, examples.md.
