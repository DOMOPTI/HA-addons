# Specs (implementation plans)

Implementation plans produced by Claude Code. Each plan is a markdown file describing a feature or fix in detail, structured for a less capable executor to follow.

Naming convention: <short-slug>-<YYYY-MM-DD>.md or <short-slug>.md for current work.

After a plan is fully implemented, it stays here as historical record. Newest is most relevant for active work; older ones serve as a corpus of plans that worked for future planning sessions to learn from.
